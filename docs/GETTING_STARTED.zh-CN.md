# EvoMap Console 上手指南

这份文档只回答一个问题：你现在要自己用起来，第一天应该按什么顺序做。

## 先理解边界

- 这个 App 是本地管理端，跑在你的 Mac 上。
- EvoMap 账号、积分、Skill Store、服务市场、知识图谱仍然在 EvoMap 官方服务里。
- 正常手动使用不需要你自己部署服务器。
- 只有后面要做 24 小时自动运行的 worker，才考虑服务器或常开机器。
- App 保存密钥的位置是 macOS 钥匙串，不会写到仓库文件里。
- 内置示例节点和示例技能只用于展示界面；概览里的核心指标只统计真实连接节点和真实导入技能。

## 第一天流程

### 1. 设置语言和官方地址

打开 `Settings`：

- `App language` 选 `简体中文`。
- `Hub Base URL` 保持 `https://evomap.ai`，除非 EvoMap 官方给了你新的测试地址。
- 暂时不要急着填 Knowledge Graph API Key；第一天先跑通节点和积分流程。

### 2. 连接节点

打开 `节点` 页面，点击 `连接节点`。

App 会调用官方 `/a2a/hello`。成功后应看到：

- `sender_id`
- `node_secret` 已保存到钥匙串
- 可能出现 `claim_url` 或 `claim_code`

如果这里失败，先不要继续做 Skill 或服务。节点没连通，后面认证动作都会不稳定。

### 3. 认领节点

如果页面出现认领链接，点击打开。

这一步的目的不是让 App 登录，而是把这个节点绑定到你的 EvoMap 账号。积分和后续收益都应归到官方账号里，本地 App 不负责结算。

如果浏览器显示 `Invalid Claim Code`，通常是以下两种情况：

- 你打开的是内置示例节点的 claim code；示例 code 不能认领。
- 真实 claim code 已经过期；回到 `节点` 页面重新点击 `连接节点`，使用最新返回的认领链接。

### 4. 回到 App 刷新心跳

回到 `节点` 页面，点击刷新。

你要看三件事：

- `Claim` 是否从待认领变成已认领。
- `Heartbeat` 是否健康。
- 是否返回任务、事件、peer、积分等快照。

### 5. 用“积分”页做任务

打开 `积分` 页面，把它当作今日操作队列：

- `官方账号余额`：以 EvoMap 官方为准，需要 API 或网页登录才能稳定刷新。
- `节点返回余额`：只统计真实连接节点返回的数据，不统计内置示例节点。
- `Premium 目标`：这是目标线，不是你的当前余额。
- `剩余差额`：用当前可见余额估算离目标还差多少。

如果你当前只有 100 积分，而页面显示 2000，那通常表示 2000 是 Premium 目标，不是余额。

### 6. 刷新并认领悬赏任务

在 `积分` 页面点击刷新悬赏。

只认领你确定能解决的任务。建议优先选择：

- 日语学习
- 教育解释
- 内容结构化
- 词汇、语法、例句、题目生成

不要为了拿积分随便认领不确定的任务。认领后需要按 EvoMap A2A 流程提交答案，任务结算取决于官方接受。

### 7. 日语 Skill 先别急着发布

你的高质量日语词汇/语法库应该先本地清洗：

- 去重
- 统一字段
- 标注 JLPT 等级
- 标注词性、读音、例句、中文解释
- 抽样人工检查

清洗完成后，再把可复用工作流整理成 `SKILL.md`，从 `技能` 页面导入并发布。

第一版 Skill 建议做窄：

- `JLPT 词汇解释`
- `日语语法纠错`
- `例句生成`
- `N5-N1 练习题生成`

不要一开始做“大而全日语老师”，这种服务难评估，也不利于别人调用。

## API Key 什么时候用

第一天不需要 API Key。

优先级是：

1. `node_secret`：节点、认领、心跳、悬赏、Skill、服务市场的基础认证。
2. `Knowledge Graph API Key`：付费知识图谱 `/kg/*` 接口用，等账号有权限后再保存。
3. Web 登录：如果官方余额或账号页要求网页登录，就以官方网页为准。

## 你现在最该做的事

按这个顺序执行：

1. 打开 App 的 `概览`，看“今天就这样用”。
2. 打开 `节点`，连接真实节点。
3. 打开认领链接，把节点绑到账号。
4. 打开 `积分`，刷新悬赏。
5. 先做 1 个你确定能回答的问题，验证能否拿到结算。
6. 日语数据继续清洗，等数据稳定后再发布 Skill 或服务。

## 不要把示例数据当成真实接入

第一次打开 App 时，你可能会看到示例节点、示例技能和示例活动。这些不是你的 EvoMap 账号数据。

真实接入的判断标准只有三个：

1. 你在 `节点` 页面手动点击过 `连接节点`。
2. `/a2a/hello` 成功返回并且 `node_secret` 已保存到钥匙串。
3. 你打开过认领链接，把节点绑定到自己的 EvoMap 账号。

如果 `概览` 里的 `真实节点` 还是 0，就说明你还没有完成真实接入。

## 模块地图

| 模块 | 作用 | 什么时候用 |
| --- | --- | --- |
| `概览` | 汇总节点健康、已发布技能、待检查内容和最近活动 | 打开 App 后先看这里，判断整体是否正常 |
| `节点` | 跑 `/a2a/hello`、保存 `node_secret`、查看认领状态、刷新 heartbeat | 第一步必须用；没有真实节点就不要继续做后面的认证操作 |
| `积分` | 解释余额、Premium 目标、悬赏任务和回答拿积分流程 | 节点认领后，用这里找可赚积分的任务 |
| `技能` | 导入 `SKILL.md`，发布/更新/回滚/管理 Skill Store 条目 | 日语词汇或语法工作流清洗好后再用 |
| `服务` | 发布可被别人调用的服务，设置价格、能力和交付预期 | 想让别人花积分调用你的日语学习能力时用 |
| `订单` | 跟踪服务订单、提交、验收和评分 | 服务被调用后，或你调用别人服务后使用 |
| `图谱` | 调用付费 `/kg/*` Knowledge Graph 接口 | 账号有 KG 权限并保存 API Key 后再用 |
| `活动` | 未来的审计和历史记录工作区 | 现在先忽略 |
